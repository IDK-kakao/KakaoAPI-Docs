Pattern Scanning API ​

Find code signatures and memory patterns in Unity games.

Overview ​

Pattern scanning helps you locate functions and data structures in memory when you don't have exact addresses. This is essential for game modding where addresses change between versions.

Basic Usage ​

cpp
#include "KakaoAPI.h"

// Find a specific code pattern
uintptr_t address = Kakao::Pattern::Scan(
    "\x48\x89\x5C\x24\x08\x48\x89\x74\x24\x10",  // Pattern bytes
    "xxxxxxxxxx"                                    // Mask (x = exact, ? = wildcard)
);

if (address) {
    Kakao::Console::LogSuccess("Found at -  0x" + std::to_string(address));
} else {
    Kakao::Console::LogError("idk i dont see :/");
}

Pattern::Scan() ​

Scan the current module for a byte pattern.

cpp

1

Parameters ​

  • pattern: Byte array to search for
  • mask: Pattern mask (x = exact match, ? = wildcard)

Pattern Creation ​

Finding Patterns with Tools ​

Use tools like x64dbg, Cheat Engine, or IDA Pro to find unique byte sequences:

  1. Disassemble the target function
  2. Look for unique instruction sequences
  3. Convert to byte pattern
  4. Create appropriate mask

Example: Function Hook Pattern ​

cpp
// Original assembly:
// 48 89 5C 24 08    mov [rsp+8], rbx

Advanced Usage ​

Pattern with Offset Calculation ​

cpp
uintptr_t FindStringReference(const std::string& targetString) {
    // Find lea instruction referencing the string
    uintptr_t lea = Kakao::Pattern::Scan(
        "\x48\x8D\x15\x00\x00\x00\x00",  // lea rdx, [rip+offset]
        "xxx????"
    );
    
    if (!lea) return 0;
    
    // Calculate actual string address
    int32_t offset = *(int32_t*)(lea + 3);  // Get offset from instruction
    uintptr_t stringAddr = lea + 7 + offset;  // RIP + instruction_size + offset
    
    // Verify it's our target string
    std::string found((char*)stringAddr);
    if (found == targetString) {
        return stringAddr;
    }
    
    return 0;
}

Multi-Step Pattern Search ​

cpp
uintptr_t FindPlayerUpdateMethod() {
    // Step 1: Find string "Player"
    uintptr_t playerStr = FindStringReference("Player");
    if (!playerStr) return 0;
    
    // Step 2: Find references to this string
    std::vector<uintptr_t> refs = Kakao::Pattern::ScanAll(
        (char*)&playerStr, "xxxxxxxx"  // Look for pointer to string
    );
    
    // Step 3: Find Update method near these references
    for (uintptr_t ref : refs) {
        uintptr_t updateMethod = Kakao::Pattern::ScanRegion(
            ref - 0x1000, 0x2000,  // Search around reference
            "\x48\x89\x5C\x24\x08",  // Common function prologue
            "xxxxx"
        );
        
        if (updateMethod) return updateMethod;
    }
    
    return 0;
}

Pattern Validation ​

cpp
bool ValidatePattern(uintptr_t address) {
    if (!address) return false;
    
    // Check if address is in valid memory region
    MEMORY_BASIC_INFORMATION mbi;
    if (VirtualQuery((void*)address, &mbi, sizeof(mbi)) == 0) {
        return false;
    }
    
    // Check if memory is executable
    if (!(mbi.Protect & (PAGE_EXECUTE | PAGE_EXECUTE_READ | PAGE_EXECUTE_READWRITE))) {
        return false;
    }
    
    return true;
}

// Usage
uintptr_t addr = Kakao::Pattern::Scan(pattern, mask);
if (ValidatePattern(addr)) {
    // Safe to use address
}

Error Handling ​

cpp
uintptr_t SafePatternScan(const char* pattern, const char* mask) {
    try {
        uintptr_t result = Kakao::Pattern::Scan(pattern, mask);
        
        if (!result) {
            Kakao::Console::LogError("Pattern not found");
            return 0;
        }
        
        if (!ValidatePattern(result)) {
            Kakao::Console::LogError("Invalid pattern result");
            return 0;
        }
        
        return result;
    }
    catch (...) {
        Kakao::Console::LogError("Pattern scan exception");
        return 0;
    }
}

Best Practices ​

1. Use Unique Patterns ​

cpp
// Good: Specific and unique
const char* good_pattern = "\x48\x89\x5C\x24\x08\x48\x89\x74\x24\x10\x57";
const char* good_mask = "xxxxxxxxxxx";

// Bad: Too generic, many matches
const char* bad_pattern = "\x48\x89";
const char* bad_mask = "xx";
